<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Platform account to Trust account | Cross-account IAM role assumption for peering acceptance, Trust route updates, and DNS config |
| ECS task to Trust VPC | Egress traffic from Platform private subnets to Trust RDS (5432), SFTP (22) |
| ECS task to AWS services | HTTPS (443) to VPC endpoints for Secrets Manager, KMS, ECR, CloudWatch Logs |
| CDK context params to NetworkStack | Trust VPC ID and route table IDs injected at synth time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Elevation of Privilege | Peering-accepter IAM role in Trust account | mitigate | Role scoped to exactly `ec2:AcceptVpcPeeringConnection`, `ec2:CreateRoute`, `ec2:DeleteRoute`, `ec2:ModifyVpcPeeringConnectionOptions` — no `ec2:*` or `*`. Trust relationship limited to Platform account root. |
| T-02-04 | Elevation of Privilege | AwsCustomResource Lambda IAM policy | mitigate | Use `AwsCustomResourcePolicy.from_statements` with explicit `sts:AssumeRole` on the single peering-accepter role ARN — never `from_sdk_calls` for cross-account calls. |
| T-02-05 | Tampering / Exfiltration | ECS task security group egress | mitigate | `allow_all_outbound=False` on ECS SG. Explicit egress rules: 5432 to 10.0.0.0/16 (RDS), 22 to 10.0.0.0/16 (SFTP), 443 to 10.10.0.0/16 (VPC endpoints only). No 0.0.0.0/0. Test `test_ecs_sg_deny_all_outbound` enforces this. |
| T-02-06 | Spoofing | Peering CIDR scope | mitigate | Routes locked to exactly `10.0.0.0/16` (Trust) and `10.10.0.0/16` (Platform) per D1. No `0.0.0.0/0` in peering routes. |
| T-02-07 | Denial of Service | AwsCustomResource install_latest_aws_sdk | mitigate | Set `install_latest_aws_sdk=False` on all AwsCustomResource instances. Default `True` downloads SDK at Lambda cold start — fails in private VPC without NAT, causes timeout. |
| T-02-08 | Tampering | Orphaned Trust routes on destroy | mitigate | Every Trust route AwsCustomResource has `on_delete=AwsSdkCall(action="deleteRoute", ...)` to clean up Trust route tables on `cdk destroy`. Prevents stale routes blocking future peering. |
| T-02-09 | Information Disclosure | VPC endpoint private DNS | accept | Interface endpoints with `private_dns_enabled=True` override public DNS for service hostnames within VPC. This is desired behavior (keeps traffic on private network). CDK Vpc L2 sets `enableDnsSupport=true` and `enableDnsHostnames=true` by default. |
| T-02-10 | Spoofing | CDK context param injection | mitigate | Validate all required context params at top of `NetworkStack.__init__` with explicit `ValueError` if missing. Fail fast at synth, not at deploy. Strip whitespace from comma-separated route table IDs. |
</threat_model>
