# ADR-016: Ephemeral Infrastructure Pattern

## Date

2026-05-30

## Status

Accepted

## Context

Access-IQ is a portfolio project developed in roughly 8-hour weekly sessions. Leaving AWS infrastructure running continuously would cost $400-600/month — unjustifiable for a project with zero production traffic. The two-account architecture (Trust + Platform) compounds the problem: RDS, Transfer Family, ECS, Redshift, and NAT gateways each incur hourly charges even when idle.

Key cost drivers if left running:

| Service               | Hourly         | Monthly (always-on) | Monthly (8h/week) |
| --------------------- | -------------- | ------------------- | ----------------- |
| RDS db.t3.micro       | $0.018         | ~$13                | ~$0.58            |
| Transfer Family SFTP  | $0.30          | ~$216               | ~$9.60            |
| NAT Gateway           | $0.045         | ~$32                | ~$1.44            |
| Redshift 8 RPU        | $0.36 (active) | ~$260               | ~$2.88            |
| ECS Fargate (3 tasks) | $0.05          | ~$36                | ~$1.60            |
| **Total**             |                | **~$557**           | **~$16**          |

Redshift Serverless auto-pauses when idle, but its minimum resume cost per cold-start is non-trivial. All other services charge continuously.

## Decision

All infrastructure deploys from scratch at the start of a working session (`make up`) and is fully destroyed at the end (`make down`). A Lambda-backed budget alarm (BudgetStack) provides a safety net — if monthly spend reaches 80% of the ceiling ($10 dev / $20 prod), ephemeral stacks are automatically torn down.

### What persists between sessions

| Resource                                  | Why it persists                                           | Cost when idle          |
| ----------------------------------------- | --------------------------------------------------------- | ----------------------- |
| S3 data lake (Bronze/Silver/Gold Parquet) | Re-ingestion takes 10+ minutes; Parquet is cheap to store | ~$0.02/month            |
| ECR repository                            | Docker image rebuilds take 2-3 minutes                    | Free (within free tier) |
| Glue Data Catalog                         | Redshift Spectrum schema definitions                      | Free                    |
| Secrets Manager entries                   | Credential rotation is expensive to re-seed               | ~$0.40/month            |

### What is destroyed and recreated each session

| Resource                       | Recreate time | Why not persistent                                                 |
| ------------------------------ | ------------- | ------------------------------------------------------------------ |
| RDS Postgres (Trust)           | 4-6 min       | $13/month idle; data is synthetic and re-seeded from the simulator |
| Transfer Family SFTP (Trust)   | 1-2 min       | $216/month idle; appointment drops are re-generated                |
| VPC + NAT + peering            | 3-5 min       | $32/month idle; stateless networking                               |
| ECS cluster + tasks            | 1-2 min       | $36/month idle; Docker image persists in ECR                       |
| Redshift Serverless            | 2-3 min       | Snapshots allow restore; RPU charges on idle resume                |
| Observability (CloudWatch)     | <1 min        | Log groups are ephemeral; dashboards recreate from CDK             |
| BudgetStack (Platform + Trust) | <1 min        | Account-level, no persistent state needed                          |

### Data persistence strategy

- **S3 Bronze**: Append-only Parquet, partitioned by `ingest_date/run_id`. Survives teardown. Re-ingestion is idempotent (manifest-based skip).
- **Redshift Silver/Gold**: Destroyed with the workgroup. Rebuilt via `dbt run` (3-5 minutes) from Bronze data in S3 via Spectrum. Snapshot restore is available as an optimisation but not required — dbt is the source of truth.
- **Trust source data**: Synthetic, generated by the northshire-hospital-sim seed scripts. Full re-generation takes 2-3 minutes. No real patient data to preserve.

### Session orchestration

```
make up    → Trust bootstrap → Platform CDK → Redshift pre-warm → Trust routes → Secrets → Docker → dbt → Prefect  (20-35 min)
make down  → Prefect stop → Platform destroy → Trust budget destroy → Trust destroy  (6-10 min)
```

`session.sh` automates the full sequence including cross-account credential switching, VPC peering handshake, and Redshift Spectrum schema creation.

## Consequences

- Monthly cost for 8h/week usage: ~$16 (vs ~$557 always-on). **97% reduction**.
- Session startup latency: 20-35 minutes. Acceptable for weekly development sessions; would not work for a team needing instant access.
- No state loss risk for analytical data: S3 is the durable layer, everything else is derived. dbt models are version-controlled and deterministically reproducible.
- Redshift cold-start adds 60-90 seconds on first query after `make up`. Mitigated by the pre-warm step in `session.sh`.
- Trade-off: developer must remember to `make down`. Mitigated by the BudgetStack Lambda safety net (tears down ephemeral stacks if spend threshold is breached).
- CDK snapshots and `RemovalPolicy.RETAIN` on stateful resources (S3, ECR) prevent accidental data loss during `cdk destroy`.

## Alternatives considered

- **Always-on with scheduling**: AWS Instance Scheduler or EventBridge rules to stop/start resources on a schedule. Still requires RDS and SFTP to exist (storage charges apply even when stopped). Transfer Family has no stop/start — only create/delete. Doesn't eliminate the $216/month SFTP cost.
- **Serverless-only architecture**: Replace RDS with DynamoDB, SFTP with API Gateway + Lambda. Would eliminate idle costs but the project specifically demonstrates enterprise-shaped infrastructure (RDS, SFTP, VPC peering) for portfolio credibility. A serverless-only design tells a different story to hiring managers.
- **Partial teardown**: Destroy only the most expensive resources (SFTP, Redshift) and keep RDS + networking. Saves ~$250/month but leaves ~$50 in idle costs. Adds complexity — `make up` would need conditional logic for "which resources already exist." Full teardown is simpler and more predictable.
- **LocalStack / local Docker**: Run AWS services locally for development. Doesn't demonstrate real cross-account networking, IAM, or Spectrum. Adds a parallel environment to maintain. The portfolio value is in the real AWS deployment.
